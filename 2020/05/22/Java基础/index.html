Java 中的多态

多态是指相同类型的变量在调用一个方法时呈现出多种不同的行为特征：

  • 编译时类型，由声明变量时的类型决定
  • 运行时类型，由实际赋值给变量的对象的类型决定

当一个变量的两个类型不一致，就会出现多态

对于变量a：a 的编译类型是 BaseClass，实际类型也是 BaseClass，所以调用 baseMethod() 会执行 BaseClass#baseMethod() 方法

对于变量b: b 的编译类型是 BaseClass，但是实际赋值时，给变量 b 赋值的是 SubClass 对象，所以 b 的实际类型是 SubClass

Java 中变量、代码块、构造器之间的执行顺序

Java 程序中类中各元素的初始化顺序：

  • 先初始化静态部分，再初始化动态部分
  • 先初始化父类部分，后初始化子类部分
  • 先初始化变量，再初始化代码块和构造器

总体顺序：

  1. 父类的静态成员变量（如果是第一次加载类）

  2. 父类的静态代码块（如果是第一次加载类）

  3. 子类的静态成员变量（如果是第一次加载类）

  4. 子类的静态代码块（如果是第一次加载类）

  5. 父类的普通成员变量

  6. 父类的构造器方法

  7. 父类的构造器方法

  8. 子类的普通成员变量

  9. 子类的动态代码块

  10. 子类的构造器方法

final 关键字

修饰符

final 修饰符时，类不能被继承，并且类中的所有方法都被隐式地使用 final 修饰

修饰方法

final 修饰方法时，有两个作用：

  • 方法不能被子类重写

  • 可以让方法转换为内嵌调用，提升效率

  • 修饰变量

    final 修饰的变量一旦初始化，就不能被修改。

    当 final 修饰实例变量时赋初值，也可以在动态代码块，或构造器中赋初值（只能赋一次）

    如果 final 变量在声明时就指定了初始值，并且在编译时可以确定下来，那么编译时，final 变量本质上会变成一个宏变量，所有用到改变量的地方都直接替换成改变量的值

Integer 类会进行缓存

  • 通过 new Integer() 创建Integer 对象，每次返回全新的 Integer 对象
  • 通过 Integer.valueOf() 创建 Integer 对象，如果值在 -128 到 127 之间，会返回缓存的对象（初始化时）

实现原理

Integer 类中有一个静态内部类 IntegerCache，在加载 Integer 类时会同时加载 IntegerCache 类，IntegeCache类的静态代码块中会创建值为 -128 到 127 的 Integer 对象，缓存到 cache 数组中，之后调用 Integer#valueOf 方法时，判断使用有缓存的 Integer 对象，有则返回，无则调用 new Integer() 创建

抽象类

特点：

  1. 可以包含抽象方法

    抽象方法不提供抽象方法的实现，由集成抽象类的子类实现抽象方法

  2. 不能实例化

    抽象类不能被实例化。抽象类的构造器不能用于创建实例，是仅提供给子类调用的。除此以外，抽象类跟普通类一样，可以拥有成员变量、普通方法、构造器、初始化块、内部类、枚举等

  3. 抽象类可以被继承

    继承抽象类的子类如果实现了所有抽象方法，那么可以作为普通类使用，否则也只能作为抽象类

String, StringBuffer 和 StringBuilder

  1. 可变性

    String 是一个不可变类，任何对 String 改变都是会产生一个新的 String 对象，所以 String 类是使用 final 来进行修饰的

    StringBuffer 和 StringBuilder 是可变类，对应的字符串的改变不会产生新的对象

  2. 执行效率

    当频繁对字符串进行修改时，使用 String 会生成一些临时对象，对一些附加操作，执行效率较低

  3. 线程安全

    StringBuffer 的读写方法都使用了 synchronized 修饰，同一时间只有一个线程进行操作，所以是线程安全的，而 StringBuffer 不是线程安全的

Object 类自带方法

registerNatives

首先 Object 类有一个本地方法 registerNatives()，会在类加载时调用，主要是将 Object 类中的一些本地方法绑定到指定的函数中去，便于之后调用。

getClass

getClass() 方法会返回对象的运行时类

有一个类 Son，继承于 Father 类，instance 的编译类型是 Father，实际赋值是一个 Son 对象，所以调用 instance.getClass() 获取的是 instance 对象的运行时类心，打印的类名是”com.test.Son”

Class.forName

与 getClass() 类似的方法还有两个，一个是 Class类中的 forName() 方法，也是在运行时，根据传入的类名去加载类，然后返回与类关联的 Class 对象。也正是因为动态加载，在编译时可以没有这个类，也不会对类名进行校验，所有可能抛出 ClassNotFoundException 异常

类名+.class

还有一种方式是使用 类名+.class 来获取类关联的 class 对象，与上面两种不同之处在于，它返回的是编译时类型，也就是在编译时，类加载器将类加载到 JVM 中，然后初始化一个 Class 对象返回
